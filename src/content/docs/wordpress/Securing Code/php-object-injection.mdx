## Introduction

## How to secure

In general, we strongly do not recommend usage of deserialization process using `unserialize` or `maybe_unserialize` function. For more complex data, we can use other data format such as JSON. 

If `unserialize` or `maybe_unserialize` function are still need to be used, the best approach we can do to try prevent PHP Object Injection is to set `allowed_classes` parameter to `false` value on the function options parameter. This approach will not accept classes object on the deserialization process and mostly can prevent from worst impact.

Example implementation:

```php
function load_ig_data_2() {
	$data = isset( $_GET['data'] ) ? (string) $_GET['data'] : '';

	list( $hash, $value ) = explode( ':', $data, 2 );

	if ( empty( $value ) || empty( $hash ) ) {
		wp_send_json_error( 'Invalid data' );
	}

	$atts     = unserialize( base64_decode( $value ), ['allowed_classes' => false] );
	$tick     = ceil( time() / MONTH_IN_SECONDS );
	$expected = substr( wp_hash( $tick . $value ), -12, 10 );

	if ( ! hash_equals( $expected, $hash ) ) {
		wp_send_json_error( 'Invalid hash' );
	}


	wp_send_json_success( "testing success" );
}

add_action( 'wp_ajax_nopriv_load_ig_data', 'load_ig_data_2' );
```
